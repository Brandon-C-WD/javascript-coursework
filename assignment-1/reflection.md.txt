This assignment helped me better understand JavaScript primitive data types and how values are stored and used within a program. I learned that JavaScript has several primitive types including number, string, boolean, undefined, and null. Although undefined and null may seem similar at first, they serve different purposes. Undefined means a variable has been declared but has not yet been assigned a value, while null is used when a programmer intentionally wants a variable to have no value.

I also learned about type conversion and type coercion. Type conversion happens when a value is explicitly changed from one type to another using functions such as String(), Number(), or Boolean(). During the assignment I converted numbers to strings, strings to booleans, booleans to numbers, and null to a number. One interesting result was that converting null to a number produces 0. Learning about this behavior helped me understand how JavaScript handles different data types behind the scenes.
